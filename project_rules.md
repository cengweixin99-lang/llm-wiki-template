# LLM Wiki 项目规则

## 核心理念

**"一次整理、持续复用、复利增长"**

这是一个基于 Karpathy llm-wiki 模式的个人知识库系统。区别于传统 RAG 每次重新检索碎片信息，LLM Wiki 让知识实现持久化积累。

**人机分工**：
- 人类：筛选资源、提问、指导方向
- LLM：整理、归档、交叉引用、矛盾检测、增量更新

---

## 目录结构

```
my-llm-wiki/
├── raw/           # 原始资源（不可修改）
├── wiki/          # 维基层（LLM 维护）
├── prompts/       # 提示词
├── templates/     # 页面模板
├── index.md       # 知识地图
├── log.md         # 编译日志
└── schema.md      # 编译规范
```

---

## 页面类型

| 类型 | 用途 |
|------|------|
| **summary** | 摘要页 - 对原始资源的高密度总结 |
| **entity** | 实体页 - 人物、组织、项目、产品 |
| **concept** | 概念页 - 核心理论与定义 |
| **comparison** | 对比页 - 多个选项的对比分析 |
| **overview** | 概览页 - 某个领域的全景图 |
| **synthesis** | 综合页 - 整合多个来源的核心洞察 |

---

## 常用命令

| 命令 | 说明 |
|------|------|
| "帮我编译 raw/xxx.md" | 编译单个原始资源 |
| "帮我编译 raw/ 下的新资料" | 批量编译 |
| "帮我完善 [[页面名]]" | 补充 stub 页面内容 |
| "帮我同步一下" | 检查同步状态 |
| "把这个答案存入 wiki" | 手动触发答案回写 |

## 核心工作流

### 1. 资源摄入

```
1. 用户添加资源到 raw/
2. LLM 读取内容
3. 创建 summary 页面
4. 提取实体和概念
5. 建立交叉引用
6. 更新 log.md
```

### 2. 增量更新

当新增资源时：
- 读取现有相关页面
- 整合新信息（不覆盖）
- 检测矛盾点
- 补充交叉引用

### 3. 答案回写（复利增长）⭐

**自动触发条件**：
- 用户问题涉及多个概念的综合分析
- 答案包含有价值的洞察或结论
- 用户明确要求保存

**回写流程**：
```
1. 判断答案是否有长期价值
2. 创建或更新相关页面
3. 添加 "## 来自问答" 章节：
   - 来源问题
   - 回答日期
   - 核心洞察
4. 建立交叉引用
5. ⚠️ 必须更新 index.md：
   - 添加到"最近问答"区域
   - 添加到对应索引（概念/对比等）
   - 更新统计数字
6. 更新 log.md
```

**回写格式**：
```markdown
## 来自问答

> 来源: 用户提问 "xxx"
> 日期: YYYY-MM-DD

(核心回答内容)
```

### 4. 矛盾检测

在 synthesis 页面中：
- 对比多个来源观点
- 标注共识点
- 标注差异点
- 检测矛盾并分析原因

---

## index.md 维护规则 ⚠️

**index.md 是知识地图，必须保持最新状态。**

### 两种更新来源

| 来源 | 更新内容 |
|------|----------|
| **raw 编译** | 更新"最近编译" + 对应索引 + 统计 |
| **问答互动** | 更新"最近问答" + 对应索引 + 统计 |

### index.md 结构

```markdown
## 📝 最近编译        ← raw 编译时更新
## 💬 最近问答        ← 问答存入时更新
## 🏢 实体索引        ← 新增实体时更新
## 💡 概念索引        ← 新增概念时更新
## ⚖️ 对比索引        ← 新增对比时更新
## 📊 知识图谱统计    ← 每次都更新
```

### 必须执行的更新

每次创建 wiki 页面后，必须：
1. ✅ 添加到对应索引（实体/概念/对比等）
2. ✅ 添加到"最近编译"或"最近问答"
3. ✅ 更新统计数字

## YAML Frontmatter 规范

```yaml
---
type: summary | entity | concept | comparison | overview | synthesis
source: raw/原始文件名.md 或 URL
status: 🟢已整理 | 🟡整理中 | 🔴待补充
last_updated: YYYY-MM-DD
---
```

---

## 同步检查流程

当用户说 "帮我同步" 时：

1. 扫描 raw/ 和 wiki/
2. 检测问题：
   - 源文件已删除 → 归档 wiki 页面
   - 未编译资料 → 提示编译
   - 断裂链接 → 创建 stub 或清理
3. 输出报告
4. 用户确认后执行清理

---

## 删除/归档规则

### 删除操作

```
1. 删除文件
2. 扫描引用该文件的页面
3. 报告断裂链接
4. 更新 log.md
```

### 归档操作

```
1. status 改为 📦已归档
2. 添加到 Archive.md
3. 保留文件
4. 更新 log.md
```
