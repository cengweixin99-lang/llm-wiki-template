# LLM Wiki 编译规范

> 基于 Karpathy llm-wiki 模式的个人知识库系统

## 1. 核心理念

**"一次整理、持续复用、复利增长"**

- 传统 RAG：每次查询都重新检索碎片信息，无积累
- LLM Wiki：LLM 维护持久化知识，知识越用越丰富

**人机分工**：
- 人类：筛选优质资源、提出关键问题、指导分析方向
- LLM：内容摘要、分类归档、交叉引用、矛盾检测、增量更新

---

## 2. 三层架构

```
my-llm-wiki/
├── raw/           # 原始资源层 - 不可修改的"事实来源"
├── wiki/          # 维基层 - LLM 生成维护的结构化内容
├── prompts/       # 提示词 - 指导 LLM 行为
├── templates/     # 页面模板 - 标准化输出格式
├── index.md       # 知识地图入口
├── log.md         # 编译日志
└── schema.md      # 本规范文件
```

---

## 3. 页面类型

| 类型 | 用途 | 模板 |
|------|------|------|
| **summary** | 摘要页 - 对原始资源的高密度总结 | `templates/summary.md` |
| **entity** | 实体页 - 人物、组织、项目、产品 | `templates/entity.md` |
| **concept** | 概念页 - 核心理论与定义 | `templates/concept.md` |
| **comparison** | 对比页 - 多个选项的对比分析 | `templates/comparison.md` |
| **overview** | 概览页 - 某个领域的全景图 | `templates/overview.md` |
| **synthesis** | 综合页 - 整合多个来源的核心洞察 | `templates/synthesis.md` |

---

## 4. 写入规范

### 4.1 YAML Frontmatter

```yaml
---
type: summary | entity | concept | comparison | overview | synthesis
source: raw/原始文件名.md 或 URL
status: 🟢已整理 | 🟡整理中 | 🔴待补充
last_updated: YYYY-MM-DD
---
```

### 4.2 双向链接

- 使用 `[[文件名]]` 格式创建链接
- 即使目标不存在也需打上链接
- 链接会在 Obsidian 图谱中显示为节点间的连线

### 4.3 结构化输出

| 元素 | 使用场景 |
|------|----------|
| 列表 | 要点、步骤、注意事项 |
| 表格 | 对比、参数、属性 |
| 代码块 | 代码示例、命令、配置 |
| Mermaid | 关系图、流程图 |
| 引用块 | 定义、重要提示 |

## 5. 核心工作流

### 5.1 资源摄入流程

```
1. 用户添加原始资源到 raw/
2. LLM 读取资源内容
3. 创建 summary 页面
4. 提取实体和概念，创建/更新对应页面
5. 建立交叉引用
6. 更新相关 overview 页面
7. 记录到 log.md
```

### 5.2 增量更新

当新增资源时：

```
1. 读取现有相关页面
2. 整合新信息到现有内容
3. 检测并标注矛盾点
4. 补充交叉引用
5. 更新 last_updated
```

**严禁直接覆盖现有内容**

### 5.3 答案回写（复利增长）

当用户提问产生优质答案时：

```
1. 判断答案是否具有长期价值
2. 如果是，创建或更新相关页面
3. 在页面末尾添加 "## 来自问答" 章节
4. 标注来源问题和日期
5. ⚠️ 必须更新 index.md（见 5.5）
6. 更新 log.md
```

### 5.4 矛盾检测

在 synthesis 页面中：

```
1. 对比多个来源的观点
2. 标注共识点
3. 标注差异点
4. 检测矛盾并分析可能原因
```

### 5.5 index.md 维护规则 ⚠️

**index.md 是知识地图，每次创建/更新 wiki 页面后必须同步更新。**

**两种更新来源：**

| 来源 | 更新内容 |
|------|----------|
| **raw 编译** | 更新"最近编译" + 对应索引 + 统计 |
| **问答互动** | 更新"最近问答" + 对应索引 + 统计 |

**index.md 结构：**

```markdown
## 📝 最近编译        ← raw 编译时更新
## 💬 最近问答        ← 问答存入时更新
## 🏢 实体索引        ← 新增实体时更新
## 💡 概念索引        ← 新增概念时更新
## ⚖️ 对比索引        ← 新增对比时更新
## 📊 知识图谱统计    ← 每次都更新
```

**必须执行的更新：**

每次创建 wiki 页面后，必须：
1. ✅ 添加到对应索引（实体/概念/对比等）
2. ✅ 添加到"最近编译"或"最近问答"
3. ✅ 更新统计数字

## 6. 状态标识

| 状态 | 含义 | 行动 |
|------|------|------|
| 🟢已整理 | 内容完整，结构清晰 | 可直接使用 |
| 🟡整理中 | 正在处理 | 等待完成 |
| 🔴待补充 | Stub 页面，仅有框架 | 需要补充内容 |

---

## 7. 同步与维护

### 7.1 同步检查

```
"帮我同步" → 检查 raw 和 wiki 的同步状态
```

检测项目：
- 源文件已删除的 wiki 页面
- 未编译的 raw 资料
- 断裂链接

### 7.2 定期维护

- 清理断裂链接
- 合并重复内容
- 更新过时信息
- 归档过期页面
